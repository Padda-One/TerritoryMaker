# Contributing to Territory Maker

Thank you for your interest in contributing. This project is maintained by a single developer and contributions are welcome — but please read this document carefully before opening any issue or pull request.

## Before you contribute

- Read the relevant source files before proposing a change. Familiarity with the code is required.
- Make sure your idea is not already listed in open Issues.
- For any non-trivial change, **open an Issue first** to discuss the approach before writing code.

## Opening an Issue

- Use the provided templates.
- Be specific: include steps to reproduce bugs, the browser you tested on, and what you expected vs what happened.
- Feature requests must include a clear use case — "it would be nice if" is not sufficient.

## Submitting a Pull Request

- Every PR must reference an existing Issue (`Closes #123`).
- PRs without a linked Issue will be closed without review.
- Keep changes focused — one concern per PR.
- No new dependencies without prior discussion in an Issue.
- Test your change manually in a browser before submitting.

## What will be rejected without review

- PRs that are not linked to an open Issue.
- PRs that contain code generated automatically by an AI tool without human review and manual testing.
- PRs that introduce large refactors without prior discussion.
- PRs with vague descriptions ("misc fixes", "improvements", etc.).
- Mass-generated or templated contributions.

## Code style

- TypeScript strict mode is enforced — no `any` without justification.
- No comments in source code unless the logic is genuinely non-obvious.
- Keep it simple. Three similar lines are better than a premature abstraction.
